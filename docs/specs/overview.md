# repiq

## What is This

AIエージェントがOSSライブラリ/リポジトリ選定を行うための、高速・非推論型データ取得CLI。客観的指標のみを取得し、構造化データとして返す。判断・推薦・スコア付けは一切行わない。

## Who is This For

- AI エージェント (Claude Code, Cursor, Copilot 等) がライブラリ選定の「思考素材」を得るために使う
- Vibe coding ワークフローに組み込む開発者
- OSS コミュニティ全般

## Problem

AIエージェントがライブラリ選定を行うとき、必要な客観データを素早く構造化された形で取得する手段がない。既存ツール (Scorecard, ORT, CHAOSS, gh CLI) はそれぞれ特定領域に特化しており、エージェント向けに統一されたデータレイヤーとして設計されていない。

## Vision

repiq は「データ正規化レイヤー」として、複数のソース (GitHub, npm, crates.io, PyPI 等) からの指標を統一スキーマで提供する。6-12ヶ月で主要なパッケージレジストリをカバーし、AI エージェントがリポジトリ/ライブラリ選定に必要な全データを repiq 一つで取得できる状態を目指す。

## Principles

- **思考しない**: 推論は AI に委譲。repiq は客観データのみを返す
- **最小主義**: 必要最小限の指標から始め、拡張は後から
- **Agent-first**: JSON/NDJSON 出力、固定スキーマ、deterministic
- **速度優先**: Vibe coding の流れを止めない (3秒以内)
- **役割分離**: `repiq -> データ取得` / `AI -> 推論` / `human -> 最終決定`
